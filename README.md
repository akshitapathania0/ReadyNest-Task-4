# CollabSpace вЂ” Real-Time Collaborative Platform

A full-stack real-time collaboration app built with React, Node.js, Socket.IO, and MongoDB. Multiple users can simultaneously edit documents, track changes, chat, and restore version history.

---

## Features

| Feature | Implementation |
|---|---|
| User Authentication | JWT-based login/register with bcrypt hashing |
| Real-time Collaboration | Socket.IO rooms with OT conflict handling |
| Multi-user Sync | Live content broadcast to all connected clients |
| Conflict Handling | Operational Transform (insert/delete transforms) |
| Activity Tracking | Active users panel, typing indicators, live cursors |
| Data Persistence | MongoDB with Mongoose; auto-save every 5s |
| Version History | Up to 50 named versions per document; restore any |
| Live Cursor Tracking | Cursor positions broadcast in real-time |
| Export | Export as .txt, .md, or .json |
| Web Notifications | Toast system for joins, saves, errors, messages |
| Dark Mode | Full dark/light theme with persistence |

---

## Tech Stack

- **Frontend:** React 18, React Router 6, Socket.IO Client, Axios
- **Backend:** Node.js, Express, Socket.IO, JWT, bcryptjs
- **Database:** MongoDB (via Mongoose)
- **Deploy:** Render (single service, Express serves React build)

## Local Development

### Prerequisites
- Node.js 18+
- MongoDB running locally OR a MongoDB Atlas connection string

### Setup

```bash
# 1. Clone and enter the project
git clone <your-repo>
cd collabspace

# 2. Install all dependencies
npm run install:all

# 3. Set up server environment
cp server/.env.example server/.env
# Edit server/.env with your MONGODB_URI and JWT_SECRET

# 4. Start the server (port 5000)
npm run dev:server

# 5. In a new terminal, start the client (port 3000)
npm run dev:client
```

Open http://localhost:3000

## Deploy to Render

### Step 1 вЂ” MongoDB Atlas
1. Create a free cluster at [mongodb.com/cloud/atlas](https://mongodb.com/cloud/atlas)
2. Create a database user and whitelist `0.0.0.0/0` (allow all IPs)
3. Copy your connection string: `mongodb+srv://user:pass@cluster.mongodb.net/collabapp`

### Step 2 вЂ” Push to GitHub
```bash
git init
git add .
git commit -m "initial commit"
git remote add origin https://github.com/YOUR_USERNAME/collabspace.git
git push -u origin main
```

### Step 3 вЂ” Create Render Service
1. Go to [render.com](https://render.com) в†’ **New** в†’ **Web Service**
2. Connect your GitHub repo
3. Render auto-detects `render.yaml` вЂ” just click **Create**
4. In the **Environment** tab, set:
   - `MONGODB_URI` в†’ your Atlas connection string
   - `CLIENT_URL` в†’ `https://your-service-name.onrender.com`

### Step 4 вЂ” Done!
After the build (~3 min), your app is live at `https://your-service-name.onrender.com`

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `MONGODB_URI` | Yes | MongoDB connection string |
| `JWT_SECRET` | Yes | Secret for signing JWTs (auto-generated by Render) |
| `CLIENT_URL` | Yes | Your Render URL (for CORS) |
| `PORT` | No | Defaults to 5000 (Render sets 10000) |
| `NODE_ENV` | No | Set to `production` on Render |

## API Reference

### Auth
- `POST /api/auth/register` вЂ” `{ username, email, password }`
- `POST /api/auth/login` вЂ” `{ email, password }`
- `GET /api/auth/me` вЂ” Get current user (requires JWT)
- `PUT /api/auth/profile` вЂ” Update username/color

### Documents
- `GET /api/documents` вЂ” List accessible documents
- `POST /api/documents` вЂ” Create new document
- `GET /api/documents/:id` вЂ” Get document
- `PUT /api/documents/:id` вЂ” Update content/title
- `DELETE /api/documents/:id` вЂ” Delete (owner only)
- `POST /api/documents/:id/collaborators` вЂ” Invite collaborator
- `DELETE /api/documents/:id/collaborators/:userId` вЂ” Remove collaborator
- `GET /api/documents/:id/versions` вЂ” Version history
- `POST /api/documents/:id/versions/:vId/restore` вЂ” Restore version
- `GET /api/documents/:id/activity` вЂ” Activity log

### Socket Events (client в†’ server)
| Event | Payload | Description |
|---|---|---|
| `join_document` | `{ documentId }` | Join a doc room |
| `content_update` | `{ documentId, content, title? }` | Broadcast content |
| `cursor_move` | `{ documentId, cursor }` | Share cursor position |
| `selection_change` | `{ documentId, selection }` | Share text selection |
| `typing` | `{ documentId, isTyping }` | Typing indicator |
| `save_version` | `{ documentId, message }` | Manually save version |
| `chat_message` | `{ documentId, message }` | Send chat message |
| `leave_document` | `{ documentId }` | Leave room |
